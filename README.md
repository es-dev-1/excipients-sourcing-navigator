# Excipient Sourcing Navigator

An interactive tool to help pharma and nutra formulators find fitting excipient suppliers, organized by route of administration, functional category, and excipient type.

## Status
Active -- kickoff 2026-05-30

## Goal
Build a web-based tool that allows users to navigate the excipient landscape by route of administration > category > excipient > supplier and quickly identify relevant suppliers for their formulation needs.

## Quick context
A single-page web tool (`index.html` plus two JSON data files) that lets PharmaExcipients.com visitors navigate the excipient landscape by route of administration > category > excipient > supplier > product. It builds on the pharma-excipients-landscape structure tree.

## Stages
1. Landing
2. Route of administration (7: Oral, Injectable, Inhalation / Pulmonary, Nasal, Topical, Ocular, Rectal & Vaginal)
3. Functional category
4. Excipient
5. Supplier (with a link to the supplier's PharmaExcipients.com page)
6. Products for that excipient + supplier (linked to the shop where a match exists)

## Data files and how to regenerate
Both JSON files are generated by `build-data.py` from the source files in `ea-work/data-for-reoccurring-tasks/`. To rebuild after any source update, run `python3 build-data.py` from this folder.

- `excipients.json` (stages 2-5): route > category > excipient > suppliers, with each supplier carrying its portal page link.
  - Source: `excipients-landscape-structure-suppliers.md` (taxonomy + suppliers per excipient).
  - Supplier names are normalized to the canonical roster in `excipient-suppliers-list.md`.
  - Supplier page links come from `supplier-page-urls.md` (`--` means no portal page, so no Supplier Page button).
- `products.json` (stage 6): supplier > products (name, chemical, function, optional shop url).
  - Source: `excipient-products-by-supplier.md`.
  - Product shop URLs are matched by exact (normalized) product name against `pe-shop-products-complete.csv`.

## Behavior notes
- Excipients with no supplier on the portal (marked `(none in supplier list)` in the structure file) show a "currently unavailable" note instead of supplier buttons.
- Products without a shop-URL match still display as non-clickable info cards. Match rate is roughly half, because the products file lists specific grades from broad web research while the shop lists generic names; exact matching avoids mis-linking grades.
- Supplier page URLs live on the data (the `link` field), not in `index.html`. The in-page `SUPPLIER_URLS` map is intentionally empty and kept only as an optional override.
